# USER_FLOWS — 關鍵使用流程

以第一位使用者（作者本人）的實際月循環為主軸。每個流程標記里程碑。

## F1 快速記帳（M1，最高頻）

**情境**：便利商店刷卡後，站在路邊 10 秒內記完。

```text
開 PWA（主畫面 icon）→ 首頁大按鈕「記一筆」
→ 金額（數字鍵盤先出）→ 選帳戶/卡（最近使用排前）→ 選分類 → 商家（選填）→ 記下
→ 商家與自己的歷史交易完全吻合時：先帶入最常用分類，顯示「依過去 n 筆帶入」，使用者仍可改
→ 樂觀寫入本機，背景同步（M2 起）
```

邊角：離線→照常完成，顯示「排隊中」（F2）；選信用卡→交易入該卡當期週期。

手機上的「從哪裡付」與「這筆錢去了哪一類」固定為單列橫向滑動，不用多排 chip 把儲存按鈕推到很下面；桌面寬度才自動換行展開。

## F2 離線記帳 → 恢復同步（M2）

```text
飛航模式記 3 筆 → 每筆入 outbox，UI 標「排隊中(3)」
→ 恢復網路 → outbox FIFO flush → 全部 applied → 標記消失
→ 若其中一筆撞版本衝突 → 該筆標「衝突」→ 點開看兩版本 → 選擇保留哪個 → 重送
```

## F3 新增信用卡與週期（M1）

```text
帳戶頁 → 新增信用卡 → 發卡行/名稱/末四碼/額度（或選共用額度群組）
→ 結帳日 15、繳款日 3、自動扣款 → 選扣款銀行帳戶
→ 完成後卡片頁顯示：本期已入帳、待入帳、上期待繳
```

邊角：共用額度群組先建群組再掛卡；月底結帳日（31）在小月自動取月底。

## F4 匯入帳單 → 審計（M3，核心流程）

**情境**：彼岸花銀行帳單 PDF 寄來，20 分鐘的人工核對變 2 分鐘的確認。

```text
PWA →「匯入帳單」→ 選檔/分享進來/貼上文字
→ importer 自動辨識（信心低→手選）→ 解析 → 建立 audit session
→ 報告頁先顯示結論：帳單 NT$ 23,456 vs 已配對 NT$ 22,251，尚有 NT$ 1,205 未對上
   ├─ 每項先顯示日期、商家、金額、差異類型與白話原因，再讓使用者決定
   ├─ 有配對候選：顯示配對信心 →「套用這組配對」或「不要配對」
   ├─ 帳單有、帳本沒有：先套用歷史分類；未知者預填「其他支出」並標「分類待複核」
   │  ├─ 可逐筆改分類後「加入帳本」
   │  └─ 或一次把所有已有分類的安全明細加入帳本（每筆仍獨立驗證與留 audit log）
   ├─ 退款缺原交易等必要事實：明示「保留未解不會修改帳本」→ 保留未解或不採用提案
   └─ 候選 ID、reasoning codes 與 evidence 收在可展開的「技術證據」中
→ 全部處理 → 顯示「平了 ✅」→ 建立 statement + 預計扣款 → 封存報告
```

邊角：中途關頁→session 保留续作；差額無法解釋→明列「未解 NT$ x」封存，不假裝平了。

一般消費的「從哪裡付」只列現金／銀行／電子支付等日常資產與信用卡，不列 `brokerage_settlement` 或 `investment_asset`；投資買賣必須從投資流程進入，不能偽裝成信用卡消費。

## F5 預計扣款確認（M1 手動 / M5 通知觸發）

```text
（M5）Discord DM：「Netflix 明天要扣 NT$ 390」
→ 扣款後：PWA（或 /finance confirm）→ 待確認清單 → 確認 → expected 轉正式交易
→ 逾期未出現 → 提醒「這期 Netflix 還沒看到扣款」
```

## F6 信用卡繳款（M1）

```text
卡片頁 →「記繳款」→ 金額（預填帳單額）→ 選付款銀行帳戶 → 存
→ 內部：轉帳分錄（銀行↓、卡債↓），不是支出 → 帳單標 paid
```

## F7 投資更新（M4）

```text
月初 → 投資頁 → 各券商：按「自動更新報價」（TWSE／Finnhub）→ 視需要更新 USD/TWD 匯率
→ 買了 ETF → 記買入：交割帳戶 −5,020（含手續費 20）→ 持倉數量/均價自動重算
→ 首頁淨資產更新，標「價格更新於今天」
```

邊角：超過 7 天未更新→首頁標示過期（M5 起發提醒）。

## F8 Discord 連結與查詢（M5）

```text
PWA 設定 → 連結 Discord → OAuth → 完成
→ 任意時刻 DM bot：/finance upcoming
→ 「未來 14 天：7/20 彼岸花信用卡扣款 NT$ 23,456、7/22 Netflix NT$ 390」（依隱私模式）
```

## F9 新裝置上線（M2）

```text
新 iPad → 開 PWA URL → Passkey 登入 → 裝置註冊（命名「iPad」）
→ 全量拉取 change feed（seq=0）→ IndexedDB 鏡像建立 → 可離線使用
→ 設定頁可看所有裝置、撤銷舊裝置
```

## F10 匯出與還原（M1 匯出 / M2 還原）

```text
設定 → 匯出 → 下載完整 JSON（+各實體 CSV zip）
→ 換伺服器：新實例 → 還原精靈 → 上傳 JSON → 驗證 → 資料完整重建
```

## F11 月循環全景（作者的實際節奏）

```text
每天：F1 隨手記帳（10 秒級）
帳單日後：F4 審計（每卡一次，2 分鐘級）
繳款日前：F5/F6（Discord 提醒 → 確認）
月初：F7 投資更新 → 首頁看淨資產全貌
隨時：F8 Discord 查詢
```

M0 之後任何 UI/功能設計，若讓上述任一流程變慢或步驟變多，需要說明理由。
