# PRODUCT_VISION — okane-dokoitta

> お金どこいった？
> 錢花去哪，打開就知道

## 1. 一句話定位

**開源、免費、可自架的個人財務審計與資產中控台。**

核心產品句：**錢花去哪，打開就知道**

英文標語：See where your money went

## 2. 解決什麼問題

記帳軟體很多，但幾乎都停在「記」。真正的痛點在「對」：

- 每月信用卡帳單來了，要花大量時間逐筆人工核對
- 金額、帳戶、卡片、日期常常記錯，錯了也不知道
- 消費日、授權日、入帳日、帳單日、扣款日是五件事，多數軟體只有一個 `date`
- 多帳戶＋多卡＋多券商，資產散落各處，沒有一個地方看得到全貌
- 不想因為帳戶數量被迫訂閱，也不想把財務資料交給封閉服務

okane-dokoitta 的核心差異是**自動財務審計**：匯入帳單，系統自動配對帳本、算出差額、指出「差在哪」——記錯、漏記、重複、延後入帳——並且每個結論都附證據，不是模糊的 AI 猜測。

## 3. 目標使用者

第一位使用者是專案作者本人。第一版以作者的完整工作流為驗收標準。

使用者輪廓：

- 個人資產規模普通的一般人、小資族
- 多個銀行帳戶、多張信用卡（部分共用額度）、多個券商（台股／美股／ETF）
- Apple 全家桶 + Windows 混用，高度使用 Discord
- 容易記錯金額、帳戶、卡片或日期
- 希望資料自己掌控：可自架、可完整匯出

**不是**目標使用者：企業、理財顧問、富裕投資人、追求投資績效排名的人。

產品不是只給作者一個人用的：一個實例要能供多位使用者同時使用，各自持有獨立帳本（隔離要求見 `docs/SECURITY.md`）。功能優先序仍以作者的工作流為準。

## 4. 產品根本原則

這些原則不得被後續功能破壞（完整規則見 `AGENTS.md` 第 3 節）：

1. **開源與免費**：核心功能永久免費，不限制帳戶、卡片、交易、匯入、匯出、自架。未來只對產生實際營運成本的選用服務收費（官方託管、官方 AI 額度、行情代理、加密備份、郵件收單）。
2. **資料所有權**：一切資料可完整匯出（JSON + CSV），不建立資料封閉。
3. **帳務正確性優先**：資料正確性 > 可追蹤性 > 安全性 > 同步穩定性 > 操作效率 > 視覺與動畫。
4. **AI 不當理財老師**：AI 解析、正規化、配對、解釋；不評分、不說教、不擅自改帳。Rule-based engine 是主要真相來源。

## 5. 品牌語氣

它應該像：對帳時陪你一起找問題的夥伴。認真處理帳務，但不說教。柔和、可愛、稍微電波。

| 可以說 | 不可以說 |
|---|---|
| 找到一筆可能少記 100 元的交易。 | 您的財務健康分數下降，建議減少非必要支出。 |
| 這筆退款找不到原始消費，要幫你補一筆嗎？ | 本月餐飲支出超標 23%。 |
| 帳單和帳本差 1,205 元，可能是這兩筆。 | 恭喜！您擊敗了 87% 的投資人。 |

禁止的視覺與文案方向：西裝、金幣、K 線牆、企業藍 FinTech 感、財富自由話術、消費羞辱、財務健康評分。

品牌名稱寫法與識別規範見 `AGENTS.md` 第 11 節。

## 6. 非目標（Non-goals）

- 不做記帳社群、消費排行榜、好友比較
- 不做投資建議、選股訊號、績效競賽
- 不做銀行帳密登入、網銀爬取、未授權的自動抓帳
- 不做企業記帳、發票開立、報稅申報
- 不追求原生 App 上架（PWA 優先；架構不封死未來包裝原生的可能）

## 7. 成功標準

第一階段（M1–M3 完成）成功的定義：

- 作者每月對帳時間從「大量人工逐筆核對」降到「確認系統列出的差異清單」
- 帳本與帳單的差額每次都能被解釋（找出原因或明確標記未解）
- 所有帳戶、卡片、投資的全貌在一頁看得到，且標明資料最後更新時間
- 資料隨時可完整匯出、自架實例可在一台機器上用 Docker Compose 跑起來
